# 项目定位

## 一句话
EasyShift-MaaS 是 LLM 驱动的迁移流水线，用来自动生成并校验工业预测-优化配置。

## 目标用户
1. 有大量现场点位和历史配置负担的工业/B 端团队。
2. 需要从 legacy YAML 和非标准命名迁移到统一模板的团队。
3. 需要可解释发布门禁和回归评估的团队。

## 核心问题
1. 点位命名混乱，字段语义不统一。
2. 目标、约束、安全边界散落在脚本中，难以复用。
3. 直接依赖 LLM 输出有幻觉风险，缺少发布前硬校验。

## v0.3 解决路径
1. Parser Agent 负责点位语义映射。
2. Generator Agent 负责草案生成。
3. Deterministic Gate 负责校验和评分阻断。
4. Critic Agent 在失败后给出修正指令。

## 预期效果
1. 减少人工改模板时间。
2. 提高草案一次通过率。
3. 提供完整失败原因和修正轨迹。
