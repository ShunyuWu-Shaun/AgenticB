# 项目定位

## 一句话
EasyShift-MaaS 提供的是 **迁移基础设施**，不是某个固定行业算法包。

## 目标用户
- 正在做工业优化/调度/能效场景的技术团队。
- 当前有大量点位、多个数据源（Redis/MySQL）和历史算法脚本。
- 需要把算法标准化、可复用、可审计、可回归。

## 核心问题
很多现场系统的问题主要集中在以下方面：
- 点位定义分散，字段语义不统一。
- 约束、目标、安全边界写在多个脚本里，迁移成本高。
- 迁移后缺少明确是否可发布的标准。

EasyShift-MaaS 的方案是：
- 用 `PointCatalog` 统一点位 -> 字段映射。
- 用 `ScenarioTemplate` 统一目标函数、约束、预测配置、安全规则。
- 用 `MigrationDraft` 表达迁移草案并附带风险与待确认项。
- 用 `TemplateQualityReport` 做分层质量判定。

## 你将如何使用它
1. 导入 YAML 点位配置（standard 或 legacy）。
2. 构建上下文（从 Redis/MySQL 拉取快照值）。
3. 生成迁移草案。
4. 校验草案与质量门禁。
5. 发布模板并进行仿真/评测。

## 预期效果（建议跟踪）
- 新场景迁移速度提升（从周缩短到天）。
- 模板可发布率提升（减少临时脚本改动）。
- 回归失败率降低（上线前问题提前暴露）。
